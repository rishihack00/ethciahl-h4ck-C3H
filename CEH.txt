🔹 What is a Network Stack?
A network stack is a collection of software components that work together to handle network communication between devices. It is responsible for sending, receiving, and processing data over a network.

Think of it like a layered system in One Piece:

Just like how a pirate crew has different roles (navigator, cook, swordsman, etc.), the network stack has layers, each handling specific tasks.
The lower layers handle physical connections (like cables or Wi-Fi), while the higher layers deal with applications (like web browsers or online games).
🔹 How Does a Network Stack Work?
A network stack follows the OSI (Open Systems Interconnection) Model or the TCP/IP Model, which breaks communication into layers:

📌 TCP/IP Model (Used in Real Networks)

Layer			Function									Example

Application Layer	Handles user applications (e.g., web browsing, email)	HTTP, FTP, DNS

Transport Layer	Manages reliable data delivery	TCP, UDP ( User datagram protocol )

Internet Layer	Routes data between networks				IP (IPv4, IPv6)

Network Access Layer	Deals with physical transmission	Ethernet, Wi-Fi

🔹 Breakdown of Each Layer


1️⃣ Application Layer (Top-most)

Handles network-related applications like web browsers, email clients, and messaging apps.
Uses protocols like HTTP, FTP, SMTP, DNS.
Example: When you open https://google.com, your browser uses the HTTP protocol.

2️⃣ Transport Layer

Ensures reliable or fast delivery of data.
Uses TCP (reliable, ordered) and UDP (fast, no guarantee).
Example: Watching a YouTube video uses UDP (faster), while downloading a file uses TCP (reliable).

3️⃣ Internet Layer

Routes data across networks using IP addresses.
Uses IP (Internet Protocol) for addressing and routing.
Example: When you send a request to Google, it finds the fastest route using IP.

4️⃣ Network Access Layer (Lowest Layer)

Handles the physical connection (Wi-Fi, Ethernet, mobile data).
Uses MAC addresses for local communication.
Example: Your laptop sends data over Wi-Fi to reach your router.
 
🔹 Why is the Network Stack Important?
✅ Enables Internet Communication – Without a network stack, your computer can’t send or receive data.
✅ Handles Errors and Retries – Ensures lost packets are resent and received in the correct order (via TCP).
✅ Supports Multiple Protocols – Allows web browsing (HTTP), file downloads (FTP), and video calls (WebRTC) to work together.
✅ Optimizes Performance – Routes traffic efficiently to minimize lag and data loss.

🔹 Example: Sending a Message on WhatsApp
1️⃣ Application Layer – WhatsApp formats the message.
2️⃣ Transport Layer – Uses TCP/UDP to ensure message delivery.
3️⃣ Internet Layer – Finds the fastest route using IP addresses.
4️⃣ Network Access Layer – Sends data over Wi-Fi or mobile data.

ANGRY IP SCANNER 

Main terminology:

Feeder - generator of IP addresses for scanning. Angry IP Scanner provides various kinds of feeders: IP Range, Random, and IP List File. You can select a feeder using the combo box next to the Start button.

Fetcher - gathers specific information about a host, e.g. ping time, hostname, open ports. Fetchers usually represent columns in the scanning results list. You can select additional fetchers by choosing "Tools->Select fetchers" from the menu.

Main terminology (continued):

Alive host - is the host, responding to pinging. These are blue in the results list.

Dead host - is the host, not responding to pinging (red in the list). However, it may still have ports open (if firewall blocks pings). In order to scan these hosts fully, check "scan dead hosts" in the Tools->Preferences dialog.

Open port - a TCP port, responding to connection attempts. Hosts with open ports are green in the results list.

Filtered port - a TCP port, not responding that it is closed (no RST packet). These ports are usually specifically blocked by firewalls for some reason.

Pinging (checking if hosts are alive):

Angry IP Scanner can use different methods for pinging hosts. You can choose between them in the Preferences dialog.

ICMP echo - is the standard method used by the 'ping' program. This one requires administrator or root privileges on most platforms. Note that some firewall software disables sending of ICMP echo reply packets, making alive hosts appear like dead.

UDP - sends UDP packets (datagrams) to one of the host's ports and sees if there is any response (either positive or negative). This is non-standard, but works without special privileges.

TCP - tries to connect to port 80 (http) on the host. This may work better than UDP for some networks, but usually it is worse.

UDP and TCP pinging most often doesn't properly detect routers or other network equipment.

PACKETS AND FRAMES : 
Packets and frames are both ways data is organized for sending between devices, but they work at different stages. A packet is a chunk of data created by a network protocol (like IP) that includes information about where it's from and where it’s going. Think of it as a packaged letter that has the address on it. 

A frame is what’s used at the physical link layer (like Ethernet) to actually send that packet over a cable or wireless connection. It wraps the packet with additional details needed for the local network to recognize and handle it, like sender and receiver MAC addresses and error-checking info.

Basically, the packet is the data you want to send with addresses and info added by higher layers, and the frame is how that packet gets physically sent over the local network.
EXAMPLE :
The messages we send over Discord are processed by the system as packets and then sent across the network as frames until it gets to the router and so on.
Yeah, imagine you are sending a letter through the mail.


The packet is like a single letter with the message and the sender's and receiver's addresses inside.,
The frame is like the envelope that holds the letter, including the postal stamp and the mailing address outside.

WHAT HAPPENS WHEN YOU MADE A DNS REQUEST ??


When you request a domain name, your computer first checks its local cache to see if you've previously looked up the address recently; if not, a request to your Recursive DNS Server will be made.
A Recursive DNS server is usually provided by your ISP , but you can also choose your own .

A Recursive DNS Server is usually given by your ISP, but you can use your own, like:
Google DNS → 8.8.8.8        
Cloudflare DNS → 1.1.1.1   
Both Google and Cloudflare DNS are themselves a recursive DNS server .
It handles all the DNS lookups for you to find website IPs. 


🔁 What is a Recursive DNS Server?
A recursive DNS server is like a helper that finds the IP address for a website on your behalf — even if it has to ask multiple other servers.

🧠 Simple Example:
You type tryhackme.com in your browser.

Your device asks the recursive DNS server:
“Hey, what’s the IP for tryhackme.com?”
If it doesn’t know the answer, it does the work:
📍 Asks Root DNS Server → "Where's .com info?"
📂 Asks TLD DNS Server (for .com) → "Where's tryhackme.com?"
🧠 Asks Authoritative Nameserver → Gets the IP
✅ Sends the IP back to you
And it remembers (caches) the answer for next time.

✅ In Short:
A recursive DNS server finds the IP for a domain by recursively querying other DNS servers — and gives you the final answer.

📌 Example Recursive DNS Servers:
Google DNS → 8.8.8.8
Cloudflare DNS → 1.1.1.1
Your ISP's DNS


This server also has a local cache of recently looked up domain names . If a result is found locally , this is sent back to your computer , and your request ends here ( this is common for popular and heavily requested services such as GOOGLE , FACEBOOK , TWITTER ).f

If the request cannot be found locally , a journey begins to find the correct answer, starting with the internet’s root DNS servers. 

The root servers act as the DNS backbone of the internet; their job is to redirect you to the correct Top Level Domain Server, depending on your request. If, for example, you request www.tryhackme.com, the root server will recognise the Top Level Domain of .com and refer you to the correct TLD server that deals with .com addresses.

The TLD server holds records for where to find the authoritative server to answer the DNS request . The authoritative server is often also known as the nameserver for the domain .

A nameserver is a special DNS server that holds the actual records (like IP address) for a domain name.

Why Important?
It’s the final source of truth for a domain’s DNS records.
It stores info like:

An authoritative DNS server is the server that is responsible for storing the DNS records for a particular domain name, and where any updates to your domain name, DNS records would be made. 

Depending on the record type, the DNS record is then sent back to the recursive DNS server, where a local copy will be cached for future requests and then relayed ( sent )back to the original client ( my device ) that made the request.

DNS records all come with a TTL ( Time to Live ) value . This value is  a number represented in seconds that the response should be saved , for locally until you have to look it up again . 

Caching saves on having to make a DNS request every time you communicate with a server ..





THE PROCESS :

🔁 What happens:
You request a domain like tryhackme.com.
Recursive DNS server finds the IP by asking other DNS servers.
It stores (caches) the result locally.
It serves that cached result to anyone who asks, as long as the TTL hasn't expired.

🕒 But after the TTL:
Once the TTL (like 300 seconds) is over:
The cached record is removed (erased) from memory.
The recursive server has to query the authoritative server again if the domain is requested.


🧠 In Simple Words:
✅ Yes, recursive DNS servers do cache DNS records locally,
❌ But they don’t keep them forever — only until the TTL expires.
The same goes for the caches that is stored in the user’s device .. It disappears once TTL expires ..

So the next time you visit the same website, your device:
Doesn't ask the DNS server again
Uses the cached IP (if still valid based on TTL)

HTTP in Detail :

What is HTTP? (HyperText Transfer Protocol)
HTTP is what's used whenever you view a website, developed by Tim Berners-Lee and his team between 1989-1991. HTTP is the set of rules used for communicating with web servers for the transmitting of webpage data, whether that is HTML, Images, Videos, etc.

What is HTTPS? (HyperText Transfer Protocol Secure)
HTTPS is the secure version of HTTP. HTTPS data is encrypted so it not only stops people from seeing the data you are receiving and sending, but it also gives you assurances that you're talking to the correct web server and not something impersonating it
When we access a website, your browser will need to make requests to a web server for assets such as HTML, Images, and download the responses. Before that, you need to tell the browser specifically how and where to access these resources, this is where URLs will help.

What is a URL? (Uniform Resource Locator)
If you’ve used the internet, you’ve used a URL before. A URL is predominantly an instruction on how to access a resource on the internet. The below image shows what a URL looks like with all of its features (it does not use all features in every request).



Scheme: This instructs on what protocol to use for accessing the resource such as HTTP, HTTPS, FTP (File Transfer Protocol).
User: Some services require authentication to log in, you can put a username and password into the URL to log in.
Host: The domain name or IP address of the server you wish to access.
Port: The Port that you are going to connect to, usually 80 for HTTP and 443 for HTTPS, but this can be hosted on any port between 1 - 65535.
Path: The file name or location of the resource you are trying to access.
Query String: Extra bits of information that can be sent to the requested path. For example, /blog?id=1 would tell the blog path that you wish to receive the blog article with the id of 1.
Fragment: This is a reference to a location on the actual page requested. This is commonly used for pages with long content and can have a certain part of the page directly linked to it, so it is viewable to the user as soon as they access the page.

SENSITIVE DATA EXPOSURE :

 Sensitive Data Exposure occurs when a website doesn't properly protect (or remove) sensitive clear-text information to the end-user; usually found in a site's frontend source code.
We now know that websites are built using many HTML elements (tags), all of which we can see simply by "viewing the page source". A website developer may have forgotten to remove login credentials, hidden links to private parts of the website or other sensitive data shown in HTML or JavaScript.


Sensitive information can be potentially leveraged to further an attacker's access within different parts of a web application. For example, there could be HTML comments with temporary login credentials, and if you viewed the page's source code and found this, you could use these credentials to log in elsewhere on the application (or worse, used to access other backend components of the site).
Whenever you're assessing a web application for security issues, one of the first things you should do is review the page source code to see if you can find any exposed login credentials or hidden links.





HTML INJECTION :

HTML Injection is a vulnerability that occurs when unfiltered user input is displayed on the page. If a website fails to sanitise user input (filter any "malicious" text that a user inputs into a website), and that input is used on the page, an attacker can inject HTML code into a vulnerable website.
Input sanitisation is very important in keeping a website secure, as information a user inputs into a website is often used in other frontend and backend functionality. A vulnerability you'll explore in another lab is database injection, where you can manipulate a database lookup query to log in as another user by controlling the input that's directly used in the query - but for now, let's focus on HTML injection (which is client-side).
When a user has control of how their input is displayed, they can submit HTML (or JavaScript) code, and the browser will use it on the page, allowing the user to control the page's appearance and functionality.

Other Components :

LOAD BALANCERS :
When a website gets lots of visitors or needs to always stay online, one server might not be enough. That’s where a load balancer comes in.

A load balancer sits in front of multiple servers. When you visit the site, your request goes to the load balancer first. It then chooses the best server to handle your request.


It uses smart rules (called algorithms) like:

Round-robin – sends each request to the next server in line, one by one.
Least connections (weighted) – sends the request to the server that’s handling the fewest people right now.

This helps:
Split the traffic so no server gets overloaded.
Keep the website running even if one server fails.







CDN (Content Delivery Networks) :
A CDN ( Content Delivery Network ) helps reduce traffic and speed up websites by storing static files ( like images , videos , Java script ,CSS ) on many servers around the world .

When a user visits your site , the CDN finds the closest server to the user and delivers the file from there – instead of from a faraway server. This makes the site load faster and reduces the load on your main server.

QUESTION : 
So does it actually take the data from the main server to the closest server to the user and then send the data to us from the closest server ?
--------------------------------------------------------------------------------------------------------------------------------------
ANS :
✅ First Time: When a file (like an image or video) is not yet on the nearby CDN server, the CDN fetches it from your main server, stores a copy, and then sends it to the user.
✅ Next Time: If someone else nearby requests the same file, the CDN already has it cached, so it delivers it instantly from the closest server — no need to ask the main server again.
So in short:
The first time = main server → CDN → user
After that = CDN (nearest server) → user
This is how CDNs save time and reduce load on the main server.

Databases :
Often websites will need a way of storing information for their users. Webservers can communicate with databases to store and recall data from them.

 Databases can range from just a simple plain text file up to complex clusters of multiple servers providing speed and resilience. 

You'll come across some common databases: MySQL, MSSQL, MongoDB, Postgres, and more; each has its specific features.






WAF (Web Application Firewall) :
A WAF sits between your web request and the web server; its primary purpose is to protect the webserver from hacking or denial of service attacks. It analyses the web requests for common attack techniques, whether the request is from a real browser rather than a bot.

It also checks if an excessive amount of web requests are being sent by utilising something called rate limiting, which will only allow a certain amount of requests from an IP per second. 

If a request is deemed a potential attack, it will be dropped and never sent to the webserver.












How Web servers work :
What is a Web Server?


A web server is a software that listens for incoming connections and then utilises the HTTP protocol to deliver web content to its clients. The most common web server software you'll come across is Apache, Nginx, IIS and NodeJS. A Web server delivers files from what's called its root directory, which is defined in the software settings. For example, Nginx and Apache share the same default location of /var/www/html in Linux operating systems, and IIS uses C:\inetpub\wwwroot for the Windows operating systems. So, for example, if you requested the file http://www.example.com/picture.jpg, it would send the file /var/www/html/picture.jpg from its local hard drive ..

Virtual Hosts ?
🖥️ A web server can host many websites (with different domain names) using something called virtual hosts.
When someone visits a website:
The browser sends the domain name (like one.com) in the HTTP request.
The web server checks its virtual host config files (just text files) to find a match.
If it matches (like one.com), it serves files from the correct folder (e.g., /var/www/website_one).
If it doesn't find a match, it shows the default website.
✅ Example:
one.com → /var/www/website_one
two.com → /var/www/website_two
🔁 There’s no limit to how many websites you can host this way — as long as your server has the resources!

Static Vs Dynamic Content ?

Static content, as the name suggests, is content that never changes. Common examples of this are pictures, javascript, CSS, etc., but can also include HTML that never changes. Furthermore, these are files that are directly served from the webserver with no changes made to them.
Dynamic content, on the other hand, is content that could change with different requests. Take, for example, a blog. On the homepage of the blog, it will show you the latest entries. If a new entry is created, the home page is then updated with the latest entry, or a second example might be a search page on a blog. Depending on what word you search, different results will be displayed.
These changes to what you end up seeing are done in what is called the Backend with the use of programming and scripting languages. It's called the Backend because what is being done is all done behind the scenes. You can't view the websites' HTML source and see what's happening in the Backend, while the HTML is the result of the processing from the Backend. Everything you see in your browser is called the Frontend.

THE FILE SYSYTEM 

The file system used in modern versions of  Windows  is the New Technology File System or simply  NTFS .
Before NTFS, there was  FAT16/FAT32 (File Allocation Table) and HPFS (High Performance File System). 
You still see FAT partitions in use today. For example, you typically see FAT partitions in USB devices, MicroSD cards, etc.  but traditionally not on personal Windows computers/laptops or Windows servers.
NTFS is known as a journaling file system. In case of a failure, the file system can automatically repair the folders/files on disk using information stored in a log file. This function is not possible with FAT.   
NTFS addresses many of the limitations of the previous file systems; such as: 
Supports files larger than 4GB
Set specific permissions on folders and files
Folder and file compression
Encryption ( Encryption File System or EFS )
If you're running Windows, what is the file system your Windows installation is using? You can check the Properties (right-click) of the drive your operating system is installed on, typically the C drive (C:\).

You can read Microsoft's official documentation on FAT, HPFS, and NTFS here . 
Let's speak briefly on some features that are specific to NTFS. 
On NTFS volumes, you can set permissions that grant or deny access to files and folders.
The permissions are:
Full control
Modify
Read & Execute
List folder contents
Read
Write

Another feature of NTFS is Alternate Data Streams ( ADS ).
Alternate Data Streams  (ADS) is a file attribute specific to Windows  NTFS  (New Technology File System).
Every file has at least one data stream ( $DATA ), and ADS allows files to contain more than one stream of data. Natively Window Explorer doesn't display ADS to the user. There are 3rd party executables that can be used to view this data, but Powershell gives you the ability to view ADS for files.
From a security perspective, malware writers have used ADS to hide data.
Not all its uses are malicious. For example, when you download a file from the Internet, there are identifiers written to ADS to identify that the file was downloaded from the Internet.

FOR EASY UNDERSTANDING ( MY NIKA EXPLAINATION )

1. File Systems (Think of it like a library system 📚)
A file system is how your computer organizes and stores files on the hard drive.
Without a file system, your computer wouldn’t know where one file ends and the next one starts.
👉 Old systems: FAT16, FAT32, HPFS
👉 Modern Windows: NTFS (New Technology File System)

2. Why NTFS is Better than FAT
FAT is like a basic notebook – you can write things, but it has limits.
NTFS is like a smart digital library – more powerful, secure, and flexible.
Here’s why NTFS is better:
Can handle big files (more than 4GB, unlike FAT32).
Can set permissions (decide who can open/edit/delete a file).
Can compress files/folders to save space.
Can encrypt files (lock them with a key).
Has journaling (keeps a log to fix problems if the computer crashes).

3. NTFS Permissions 🔒
Permissions = rules for who can do what with a file/folder.
Types of permissions:
Full Control → do anything (open, edit, delete, change permissions).
Modify → read + edit + delete.
Read & Execute → open and run files.
List Folder Contents → see what’s inside a folder.
Read → only look, can’t change.
Write → add new things or change content.
👉 Example:
Your teacher gives you “Read Only” access → you can see notes but not edit them.
If you had “Full Control” → you could edit, delete, or lock the notes.
How to check permissions in Windows:
Right-click file/folder → Properties.
Click Security tab.
Select a user → see what permissions they have.

4. Alternate Data Streams (ADS) 🌀
This is an extra hidden space inside a file.
Normally, a file has one main content stream (like the text in a Word file).
With ADS, you can hide extra data inside that same file, but Explorer won’t show it.
👉 Example:
Imagine a book 📖.
The visible text is the file itself.
ADS is like writing secret notes in invisible ink in the margins — still part of the book, but hidden from normal readers.
Uses:
Good use → Windows tags downloaded files with info (“This file came from the Internet”).
Bad use → Hackers can hide malware inside ADS so you don’t see it.
You can check ADS using PowerShell or special tools (since File Explorer won’t show it).

5. Quick Summary 🚀
FAT = old & simple. NTFS = new & powerful.
NTFS lets you:
✅ Store big files
✅ Control who can open/edit files (permissions)
✅ Compress/encrypt data
✅ Automatically repair after crashes (journaling)
✅ Hide extra info in files (ADS)

USER ACCOUNT CONTROL 
Alright Jon ⚡ let’s simplify this whole UAC (User Account Control) concept into easy words so you fully get it:

🔑 The Problem
Most home users run as Administrator (means: full power to install, delete, or change the system).
If malware infects you while you’re logged in as Administrator → it can also run as admin and damage your whole system.

🛡️ The Solution: UAC (User Account Control)
Microsoft made UAC to protect you.
Here’s how it works:
Even if you’re an administrator, when you log in → Windows doesn’t give you full power immediately.
👉 You’re running in standard user mode most of the time.
When a program wants to do something risky (like installing software, changing system settings, or modifying files in C:\Windows),
Windows stops and asks you:
“Do you really want to allow this?”
That’s the UAC prompt (the pop-up with the shield icon).

🔵 Example
You’re logged in as Admin.
You double-click an installer (setup.exe).
A shield icon on the installer means: “This action needs Admin rights.”
Windows pops up UAC prompt:
If you’re an Admin → just click Yes (or enter your password if required).
If you’re a Standard User → you must enter the Admin password to continue.
If no password is entered → the install is canceled.



⚠️ Important Notes
UAC does not apply to the built-in Administrator account (the hidden, god-level account). That account always runs with full rights.
UAC makes malware attacks harder because the program cannot silently install itself without triggering a visible prompt.

🧠 Think of UAC Like This:
Your account is like a car with turbo mode.
Normally you drive in eco mode (standard user rights).
If you want turbo → you must confirm by pressing a special button (UAC prompt).
Malware can’t secretly press the turbo button unless you allow it.

👉 So in short:
UAC = a security gate that asks for your permission before letting apps use administrator powers.
